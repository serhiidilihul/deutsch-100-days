# Правила создания уроков немецкого языка (A1-A2)

Данный документ регламентирует программу обучения, структуру, объемы, правила верстки и стилистику при создании интерактивных HTML-уроков немецкого языка. Все новые материалы должны строго соответствовать этим правилам для сохранения единого пользовательского опыта и корректной работы интерфейса.

---

## 1. Программа обучения (Curriculum)

Вся сетка занятий и учебный план зафиксированы во внешнем файле **`curriculum.pdf`**. 
Каждая запись в программе обучения строго следует структуре:
> **Номер урока** — **Тема урока** — **Ожидаемый результат** (чему студент научится к концу занятия).

*Пример соответствия:*
*   **Урок 6** — **Артикли и род существительных** — **Понимаю концепцию рода в немецком. Знаю маркеры рода (суффиксы -ung, -heit, -er). Умею использовать ein/eine при первом упоминании.**

*   **Критически важно:** Приступая к генерации или верстке очередного урока, обязательно сверяйтесь с `curriculum.pdf`. Название файла (`a1-X.html`), заголовки внутри страницы и блок «Lernziele» (Цели обучения) должны в точности отражать формулировки из этого документа.

---

## 2. Общие принципы и стиль

*   **Язык обучения:** Все грамматические и лексические объяснения пишутся на **русском языке** простым, дружелюбным и доступным языком (без перегруза академическими терминами).
*   **Двуязычные примеры:** Каждое немецкое словосочетание, пример или предложение в теории обязательно сопровождается переводом на русский язык в скобках или через тире.
*   **Сложность лексики:** Используется только базовая лексика уровней А1-А2. Предложения в примерах должны быть короткими и понятными.
*   **Гигиена раскладки клавиатуры (КРИТИЧЕСКИ ВАЖНО):** 
    > **Строго запрещено** смешивать русские и немецкие буквы в пределах одного слова. Автогенераторы часто путают схожие по написанию символы (например, русские «а», «е», «о», «с», «х» и латинские «a», «e», «o», «c», «x»). 
    > Все немецкие слова должны состоять исключительно из символов латиницы. Все русские объяснения — из кириллицы.

## 3. Структура урока

Каждый HTML-файл урока (например, `a1-6.html`) должен содержать следующие последовательные блоки:

### I. Шапка и навигация (вверху страницы)
*   Ссылка на предыдущий урок (активная или заблокированная классом `disabled`, если это первый урок).
*   Название текущего урока (например, «Урок А1-6»).
*   Ссылка на следующий урок.

### II. Разминка и повторение (Wiederholung)
*   **Минимум 2 полноценных упражнения** на повторение материала прошлых уроков.
*   **Объем:** Ровно по **12 примеров/вопросов** в каждом упражнении.
*   **Содержание:** Смешивание грамматики, лексики и фонетики предыдущих тем.
*   Скрытый блок ответов с кнопкой-переключателем `Показать ответы разминки`.

### III. Основная теория
*   **Concept (Понятная теория):** Дробление темы на 2-3 логических подраздела.
*   **Таблицы:** Обязательное использование таблиц для демонстрации склонений, спряжений, суффиксов и других системных данных.
*   **Лайфхаки и заметки:** Оформление важных исключений или подсказок в специальные блоки `.note` или `blockquote`.

### IV. Словарный запас (Wortschatz)
*   Список из **20-30 ключевых слов** урока (существительные даются строго с определенным артиклем и пометкой множественного числа, глаголы — в инфинитиве).

### V. Основная практика (Übungen)
*   **Минимум 5 разноплановых упражнений** для закрепления текущей темы.
*   **Объем:** В каждом из пяти упражнений должно быть **строго не менее 12 примеров (предложений, вопросов или пар)**.
*   **Типы заданий:** 
    1. Определение рода/суффиксов.
    2. Выбор правильного артикля/формы.
    3. Раскрытие скобок / вставка пропущенных слов.
    4. Заполнение пропусков в связном тексте или диалоге.
    5. Перевод предложений с русского на немецкий.

### VI. Интерактивные ответы (Lösungen)
*   Все ответы к основным упражнениям прячутся в один общий блок `.answers-block`.
*   Показ/скрытие блока реализуется через кнопку-тоггл с вызовом JS-функции `toggleAnswers('answers-main')`.

### VII. Подвал и навигация (внизу страницы)
*   Дублирование навигационных ссылок для удобного перехода на следующий шаг без скролла наверх.

## 4. Правила верстки (HTML и CSS)

Для обеспечения адаптивности и единого дизайна **строго запрещено** писать инлайновые стили (`style="..."`) или подключать новые внешние CSS-файлы. Нужно использовать только готовую сетку классов из общего файла `styles.css`:

| CSS-класс | Где используется |
| :--- | :--- |
| `.container` | Главная обертка всей страницы. |
| `.nav` / `.nav-bottom` | Блоки верхней и нижней навигации. |
| `.nav-link` | Ссылки переходов между уроками. |
| `.nav-link.disabled` | Класс для отключения ссылки (например, «Назад» на 1-м уроке). |
| `.card` | Белая карточка-подложка для каждого логического блока (Разминка, Теория, Практика). |
| `.subtitle` | Дополнительное пояснение под заголовком. |
| `.note` / `blockquote` | Выделение важной информации, исключений или цитат в теории. |
| `.vocabulary-list` | Двухколоночный или списочный блок для словаря. |
| `.vocabulary-item` | Отдельная строка в словаре. |
| `.btn-toggle` | Кнопка открытия/закрытия ответов. |
| `.answers-block` | Скрытый по умолчанию контейнер с правильными ответами. |

### JS-скрипт для интерактивных ответов
В конце каждого HTML-файла перед закрывающим тегом `</body>` должен быть размещен стандартный скрипт:

```html
<script>
    function toggleAnswers(id) {
        const block = document.getElementById(id);
        const btn = document.getElementById('btn-' + id);
        if (block.style.display === "none" || !block.style.display) {
            block.style.display = "block";
            btn.textContent = "Скрыть ответы";
            btn.classList.add("active");
        } else {
            block.style.display = "none";
            btn.textContent = id.includes('r') ? "Показать ответы разминки" : "Показать ответы урока";
            btn.classList.remove("active");
        }
    }
</script>
